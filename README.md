# Day-60-EC2

###  Overview

**Amazon EC2 (Elastic Compute Cloud)** is a web service provided by AWS that allows users to run **virtual servers (instances)** in the cloud.

It enables developers to deploy applications without managing physical hardware and provides **scalable computing capacity** on demand.

---

##  Key Concepts

###  EC2 Instance

An **EC2 instance** is a virtual machine in AWS that can run applications, websites, and data processing jobs.

Example uses:

* Web hosting
* Backend servers
* Data processing tasks

---

###  Instance Types

EC2 provides different instance types based on workload needs.

Common categories:

* **General Purpose** – balanced performance
* **Compute Optimized** – CPU-intensive tasks
* **Memory Optimized** – large data processing
* **Storage Optimized** – high disk performance

---

###  Security Groups

Security groups act as **virtual firewalls** controlling inbound and outbound traffic to EC2 instances.

Example rules:

* Allow SSH (port 22)
* Allow HTTP (port 80)

---

###  AMI (Amazon Machine Image)

An **AMI** is a preconfigured template used to launch EC2 instances.

It includes:

* Operating system
* Application software
* Configuration settings

---

##  Common Use Cases

* Hosting web applications
* Running backend services
* Big data processing
* Machine learning workloads
